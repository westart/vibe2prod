# Security Policy

## Supported versions

| Version | Supported |
| ------- | --------- |
| 0.1.x   | yes       |

## Reporting a vulnerability

Please do **not** open a public issue for security problems.

- Preferred: [GitHub private vulnerability reporting](https://github.com/westart/vibe2prod/security/advisories/new)
- Alternatively: email `security@systemconfig.it`

You can expect an acknowledgement within 72 hours. Once a fix is released,
the report and credit (if wanted) are published in the changelog.

## Scope notes

This project provisions and hardens real servers. Reports about the default
configuration shipping an insecure setting (open port, weak TLS, missing
hardening) are very much in scope — that configuration IS the product.
